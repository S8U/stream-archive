# Stream Archive 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 프로젝트명
Stream Archive - 다중 플랫폼 스트리밍 녹화 시스템

### 1.2 목적
여러 스트리밍 플랫폼(치지직, 트위치, 아프리카TV, 유튜브)의 방송을 자동으로 감지하고 녹화하여 개인 아카이브를 구축하는 웹 애플리케이션

### 1.3 지원 플랫폼
- 치지직 (Chzzk)
- 트위치 (Twitch)
- 아프리카TV (SOOP)
- 유튜브 (YouTube)

## 2. 사용자 요구사항

### 2.1 사용자 유형
- **관리자**: 채널, 녹화 스케줄, 다시보기 관리
- **사용자**: 다시보기 시청자

### 2.2 주요 기능 요구사항

#### 2.2.1 채널 관리
- **채널 생성**: 사용자는 임의의 이름으로 채널을 생성할 수 있음
- **플랫폼 연동**: 하나의 채널에 여러 플랫폼의 스트리머를 연동할 수 있음
- 플랫폼별 채널 ID 입력을 통한 연동
- 연동 시 해당 플랫폼에서 채널 정보를 자동으로 불러와 프로필 이미지 업데이트
- **접근 권한 설정**: 채널별로 공개 범위 설정
- `public`: 누구나 접근 가능
- `unlisted`: 링크를 아는 사용자만 접근 가능
- `private`: 관리자만 접근 가능

#### 2.2.2 녹화 스케줄링
- **스케줄 유형**:
- `한번만`: 특정 날짜/시간에 1회 녹화
- `매번`: 방송이 시작될 때마다 자동 녹화
- `매주 n요일`: 특정 요일에만 녹화
- `날짜 지정`: 사용자가 지정한 특정 날짜들에만 녹화
- **채널+플랫폼 단위**: 각 채널의 플랫폼별로 독립적인 스케줄 설정 가능

#### 2.2.3 자동 녹화 시스템
- **방송 감지**: 각 플랫폼의 API를 주기적으로 폴링하여 방송 시작 감지
- **중복 녹화 방지**: 스트리밍 ID를 사용하여 동일한 스트리밍에 대해 이미 녹화 중인지 DB를 통해 확인
- **녹화 기술 스택**: streamlink → 파이프 → ffmpeg를 통한 HLS 형태 녹화
- **실시간 시청**: 녹화 중에도 HLS 스트림을 통해 실시간 시청 가능

#### 2.2.4 다중 데이터 수집
1. **동영상 녹화**
- HLS 형태로 저장하여 녹화 중 실시간 시청 지원

2. **채팅 기록**
- WebSocket 연결을 통한 실시간 채팅 수집 및 저장

3. **시청자 수 기록**
- 30초 간격으로 방송 정보를 조회하여 시청자 수 변화 추적

4. **방송 제목 변경 이력**
- 30초 간격으로 방송 정보를 조회하여 제목 변경 시점과 내용 기록

#### 2.2.5 다시보기 시스템
- **녹화된 콘텐츠 재생**: 웹 브라우저에서 HLS 스트림 재생
- **채팅 동기화**: 영상 재생과 동기화된 채팅 표시
- **메타데이터 표시**: 시청자 수, 제목 변경 이력 등 부가 정보 제공
- **접근 권한**: 각 녹화 영상별로 독립적인 공개 범위 설정

## 3. 시스템 플로우

### 3.1 기본 사용 시나리오
```
1. 사용자가 '녹두로' 채널 생성
2. '녹두로' 채널에 치지직 채널 연동
 - 치지직 채널 ID: 6e06f5e1907f17eff543abd06cb62891 입력
 - 시스템이 치지직 API를 통해 채널 정보 조회 및 프로필 이미지 업데이트
3. '녹두로' 채널에 "매번 녹화" 스케줄 생성
4. 백그라운드 서비스가 치지직 API를 주기적으로 폴링
5. 방송 시작 감지 시:
 - 스트리밍 ID 확인
 - DB에서 해당 스트리밍의 녹화 상태 확인
 - 녹화 중이 아닐 경우 새로운 녹화 세션 시작
6. 방송 종료 시 녹화 자동 종료
```
