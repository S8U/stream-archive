# StreamArchive

## 개요

여러 플랫폼의 스트리밍을 녹화하고 다시보기를 제공하는 사이트

## 기능

    * [x] 플랫폼 연동 (프로필 연동)
    * [x] 공개 범위
        * [x] 공개
        * [x] 비공개
        * [x] 일부 공개 (Unlisted)
    * [x] 채널별 다시보기 리스트
2. 스트리밍 감지
   * [x] 스트리밍 정보 API Polling (10초마다)
3. 스트리밍 녹화 스케줄
    * [x] 채널별 녹화 스케줄
    * [x] 스케줄 타입
        * [x] 항상
        * [x] 한 번만
        * [x] 매주 N요일
        * [x] 날짜 선택
4. 스트리밍 녹화
    * [x] 로컬 파일 저장
    * [x] HLS 저장 (녹화 중인 스트리밍 시청 가능하도록, 세그먼트 길이 3초)
    * [x] 녹화 대상
        * [x] 동영상 (Streamlink → FFmpeg 파이프라인으로 녹화)
        * [x] 채팅 (웹소켓)

            * [x] 방송 제목 (스트리밍 정보 API Polling)
            * [x] 카테고리 (스트리밍 정보 API Polling)
            * [x] 시청자 수 (스트리밍 정보 API Polling)

        * [ ] 시간에 맞게 카테고리 변경 출력
    * [ ] 하이라이트만 보기 (채팅이 빨라지는 구간만 보여줌)

## 스트리밍 녹화 Flow

1. 스트리밍 정보 API Polling (10초마다)
2. 스트리밍이 Live 상태일 경우
    * 녹화 중이 아닐 경우
        * 채널 녹화 스케줄 확인
        * 녹화 대상일 경우
            * 녹화 정보 생성
            * 동영상 정보 생성
            * 동영상 녹화 프로세스 시작 (Streamlink → FFmpeg 파이프라인)
            * 채팅 녹화 시작 (웹소켓)
    * 이미 녹화 중일 경우
        * 동영상 메타데이터 업데이트
            * 방송 제목
            * 카테고리
            * 시청자 수
            * 썸네일 이미지 (최근 썸네일 1장, 시청자 수 제일 높을 때의 썸네일 1장)
3. 동영상 녹화 프로세스 종료 대기 (스트리밍 종료 시 Streamlink/FFmpeg 프로세스가 자동 종료됨)
4. 녹화 종료 처리
    * 채팅 녹화 종료 (웹소켓)
    * 녹화 정보 업데이트
        * 녹화 상태
    * 동영상 정보 업데이트
        * 제목 (시청자 수가 가장 많을 때의 제목으로 변경)
        * 썸네일 (시청자 수가 가장 많을 때의 썸네일로 변경)
        * 파일 크기
        * 영상 길이
